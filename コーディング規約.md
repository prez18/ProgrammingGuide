# コーディング規約

## セミコロンの有無

セミコロンは付与しない。

## 空行

メソッドとメソッドの間には空行を挿入する。
ファイルの最終行には空行を1行挿入する。  
空行は挿入する場合は複数行ではなく、1行になるように調整する。

## インデント

1インデントは半角スペース4とする。  
`Ctrl + i`で整形するか何かしらのフォーマッタで整形することを心がける。

## `:`の前後のスペースの有無

`:`の右側にのみスペースを必ずいれる。

```
// これはNG
let hoge:Hoge

// これはNG
let hoge : Hoge

// これはOK
let hoge: Hoge
```

## `,`の前後のスペースの有無

`,`の右側にのみスペースを必ずいれる。

```
// これはNG
hoge(fuga: fuga,piyo: piyo)

// これはNG
hoge(fuga: fuga , piyo: piyo)

// これはOK
hoge(fuga: fuga, piyo: piyo)
```

## `{`の扱い

`{`の左側で改行は実施しない。  
また、`{`の左側には空白を入れる。

```
// これはNG
func hoge()
{

}

// これはNG
func hoge(){

}

// これはOK
func hoge() {

}
```

## `->`の前後のスペースの有無

`->`の前後にスペースを必ずいれる。

```
// これはNG
func hoge()-> fuga {

}

// これはNG
func hoge()->fuga {

}

// これはOK
func hoge() -> fuga {

}
```

## アクセス修飾子

`internal`は明記しない。  
必要がない限り`public`を付与しない。  
外部に公開する必要がなければ`private`または`fileprivate`を付与する。

## 変数定義

ローワーキャメルケースで表現する。  
`API`などの短縮記法の場合も例外ではない。
また、mプレフィックスは付与しない。

```
// これはNG
let hogeAPI: API

// これはNG
let mRouter: Router

// これはOK
let hogeApi: API

// これはOK
let router: Router

```

## 型推論

型推論は利用できる場合は利用する。

```
// これはNG
let hoge: Int = 0

// これはOK
let hoge = 0

```

## UI系クラスの変数名

UI系クラスの変数名は接尾語として[クラス名(の一部)]を付与する。

```
// これはNG
let hogeBtn: UIButton
let fugaPageControl: UIPageControl

// これはOK
let hogeButton: UIButton
let fugaPageControl: UIPageControl
```

## イテレータ・一時的な変数

`a`, `d`, `c`, `e`, `i`, `j`, `k`, `tmp`などの命名は禁ずる。  

## Bool値の変数

`isHoge`や`hasFuga`のように真偽値であることが連想可能な命名にする。  
また、`isNotHoge`や`hasNotFuga`のような否定形の形式とならない命名にする。

## 関数・メソッド定義

関数名・メソッドはローワーキャメルケースで記述する。

## 外部引数

`hoge(fuga: fuga)`とならない命名を推奨する。  
極力、`hoge(for fuga: fuga)`など、英文法を綴るような命名とする。

## クラス定義
アッパーキャメルケースで記述する。

## 構造体定義
アッパーキャメルケースで記述する。

## プロトコル定義
アッパーキャメルケースで記述する。

## 列挙型定義
アッパーキャメルケースで記述する。  
`case ...`の定義はローワーキャメルケースとする。

## 条件分岐

原則としてif文の条件に丸括弧は使用しない。  

## 条件式

左辺を比較したい値、右辺を比較対象とする。

## switch

原則として`default`は利用しない。  
`default`を利用する場合、case式に必要な分岐を加筆し忘れたとしてもコンパイルエラーとならないため。  
また、`Int`や`String`を対象とした場合は、どうしても`default`が発生してしまうが、
そもそも、そういった対象を対象としないで済む設計にすべきである(defaultに分岐した場合のエラー処理にも困る)。

## `self`の有無

selfは必ず付与する。

## self = self

```
// Swift4.2未満を利用する場合は以下のように記述する
{ [weak self] _ in
    guard let `self` = self else {
        return
    } 
}

// Swift4.2を利用する場合には以下のように記述する
{ [weak self] _ in
    guard let self = self else {
        return
    } 
} 

```

## 改行

改行は原則として実施するものとします。  
```
// これはNG
guard let hoge = hoge else { return }

// これはOK
guard let hoge = hoge else { 
    return 
}
```

引数が複数存在する場合は原則として改行します。
```
let UseCase = UseCase(
    repository: Repository(
        localDataStore: LocalDataStore(),
        remoteDataStore: RemoteDataStore()
    )
)
```

## アンラップ

アンラップする際の名称は原則としてアンラップ前の名称と一致させます。
```
// これはNG
guard let unwrappedHoge = hoge else { 
    return 
}

// これはOK
guard let hoge = hoge else { 
    return 
}
```

## TODOFIXMEHACKXXX

何かタスクがある場合は`TODO`マーキングのみを利用する。  
`FIXME` `HACK` `XXX` などは禁止する。

## コメント

ドキュメントコメントは極力記載する。
形式は、`alt` + `command` + `/`で自動挿入されるものを利用する。
